1.0     Overview

1.1     REQ019 Introduction

        Doorstop is a requirements management tool that leverages version
        control to
        store and manage a project's documentation traced from specification
        through
        implementation.

        ![Doorstop Logo](assets/logo-black-white.png "Doorstop Logo")

2.0     Composition Features

2.1     REQ003 Identifiers

        Doorstop **shall** provide unique and permanent identifiers to linkable
        sections of text.

        Child links: LLT001, TUT001, TUT002, TUT004, TUT008

2.2     REQ004 Formatting

        Doorstop **shall** support formatting within linkable text.

        Child links: TUT001, LLT002, TUT002, TUT017, TUT019

2.3     REQ001 Assets

        Doorstop **shall** support the storage of external requirements assets.

        Child links: LLT005

2.4     REQ016 Importing content

        Doorstop **shall** be able to import content from other requirments
        management tools.

        Child links: TUT016, TUT012, TUT013

2.5     REQ017 Exporting content

        Doorstop **shall** be able to export content to interact with common
        text/document tools.

        Child links: TUT015

3.0     Presentation Features

3.1     REQ007 Viewing documents

        Doorstop **shall** provide a way to view requirements as a document.

        Child links: LLT003, TUT009, TUT010

3.2     REQ008 Interactive viewing

        Doorstop **shall** provide a way to view document items interactively.

        Child links: LLT004

3.3     REQ009 Baseline versions

        Doorstop **shall** provide the capability to baseline versions of
        requirements.

        Child links: LLT007

4.0     Administration Features

4.1     REQ011 Storing requirements

        Doorstop **shall** store requirements in a permanent and secure manner.

        Child links: TUT002, TUT004, LLT007

4.2     REQ012 Change management

        Doorstop **shall** handle change management of the requirements.

        Child links: TUT002, TUT004, LLT007

4.3     REQ013 Author information

        Doorstop **shall** associate requirements changes to existing developer
        identification.

        Child links: TUT002, TUT004, LLT007

4.4     REQ014 Scalability

        Doorstop **shall** be scalable to support thousands of requirements.

        Child links: LLT007

4.5     REQ015 Installation

        Doorstop **shall** provide a lightweight installation method.

        Child links: LLT007, LLT008
